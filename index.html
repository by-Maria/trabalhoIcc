O que é Git?

O Git é um sistema de controle de versões distribuído que permite o rastreamento de alterações em arquivos ao longo do tempo. Ele é amplamente utilizado para gerenciar o desenvolvimento de software, mas também pode ser aplicado a outros tipos de projetos.

O que é o GitHub?

O GitHub é uma plataforma baseada em nuvem que fornece hospedagem de repositórios Git. Ele permite que equipes de desenvolvimento colaborem, compartilhem e controlem as versões de seu código-fonte de forma eficiente. Além disso, o GitHub oferece recursos adicionais, como problemas, solicitações de pull e integração contínua.

Comandos GIT

A seguir, estão alguns comandos Git essenciais para você começar:

Git init

É usado para iniciar um repositório Git vazio em um diretório local. Isso cria um novo subdiretório oculto chamado ".git", onde todas as informações de controle de versão serão armazenadas.

Git clone

clona um repositório Git existente. Isso cria uma cópia local de um repositório remoto em seu computador, permitindo que você trabalhe nele e contribua com alterações.

Git branch

Cria, lista ou exclui branches (ramificações) no Git. Um branch é uma linha de desenvolvimento independente, permitindo que você trabalhe em diferentes recursos, versões ou correções simultaneamente.

Git checkout

Altera o branch atual ou restaura arquivos de um commit específico. Esse comando permite que você mude entre diferentes branches e também recupere versões anteriores de arquivos.

Git status

Mostra o estado atual do seu repositório Git. Ele exibe informações sobre arquivos modificados, adicionados ou excluídos, além de informar sobre a ramificação atual.

Git diff

Mostra as diferenças entre o estado atual do repositório e o último commit. Isso permite visualizar as alterações feitas nos arquivos antes de adicioná-las ao commit.

Git add

Adiciona arquivos ao índice do Git, preparando-os para serem incluídos em um commit. O comando "git add" permite selecionar quais alterações você deseja incluir no próximo commit.

Git commit

Grava as alterações no repositório Git como um novo commit. Um commit é um instantâneo das alterações feitas nos arquivos, permitindo que você rastreie o histórico e reverta alterações, se necessário.

Git push

Envia os commits locais para um repositório remoto. Isso atualiza o repositório remoto com suas alterações, tornando-as visíveis para outros colaboradores.

Git pull

Obtém as alterações do repositório remoto e mescla-as com o branch atual. O comando "git pull" combina o "git fetch" (obtenção das alterações) e o "git merge" (mesclagem das alterações) em um único passo.

Git revert

Desfaz um commit específico, criando um novo commit para desfazer as alterações introduzidas. Isso permite reverter commits anteriores sem perder o histórico.

Git merge

Combina as alterações de um branch com outro. O comando "git merge" é usado para incorporar as alterações de um branch secundário (ou feature branch) ao branch principal.

Git stash

salva temporariamente as alterações não finalizadas em um stash. Isso permite que você "esconda" as alterações atuais e as restaure posteriormente, permitindo uma mudança rápida de contexto.

Git rm

Remove arquivos do repositório Git e do sistema de arquivos. O comando "git rm" remove um arquivo do controle de versão do Git e também exclui fisicamente o arquivo do sistema de arquivos.

Git config

Configura opções específicas do Git. Com o comando "git config", você pode definir configurações como nome de usuário, endereço de e-mail, formatação de log, entre outros.

Git reset

Desfaz alterações em um branch, movendo o HEAD e a ramificação para um commit anterior. O comando "git reset" permite desfazer commits, descartar alterações ou mover o HEAD para um commit específico.

Git remote

Gerenciar conexões com repositórios remotos. Com o comando "git remote", você pode listar, adicionar ou remover repositórios remotos.

Git fetch

Obtém as alterações do repositório remoto, mas não as mescla com o branch atual. Isso atualiza seu repositório local com as últimas alterações, permitindo que você as incorpore posteriormente.

Git show

Exibe informações sobre commits, tags ou objetos do Git. O comando "git show" mostra detalhes sobre um commit específico, como autor, mensagem, alterações, entre outros.

Git help

Exibe informações de ajuda sobre comandos Git específicos. Você pode executar "git help" seguido do nome do comando para obter detalhes e exemplos de uso.

Git rebase

Reaplica os commits de um branch em cima de outro. O comando "git rebase" permite modificar a história do Git, movendo, combinando ou dividindo commits.

Git blame

Mostra quem modificou cada linha de um arquivo e em qual commit. O comando "git blame" é útil para rastrear a autoria de alterações específicas em um arquivo.

Git tag

Marca um commit específico com uma etiqueta. As tags são usadas para marcar versões importantes ou marcos no histórico do projeto.

Configuração e acesso ao GitHub

Siga os passos abaixo:

  1. Acesse https://github.com/ e crie uma conta no GitHub.
  2. Crie um novo repositório no GitHub.
  3. Copie o URL do repositório criado.
  4. No seu computador, navegue até o diretório do projeto usando o terminal ou o Git Bash.
  5. Execute o comando git init para inicializar um repositório Git local.
  6. Conecte o repositório local ao repositório remoto no GitHub usando o comando git remote add origin .
  7. Use os comandos Git (por exemplo, git add, git commit, git push) para adicionar e modificar arquivos no repositório do GitHub.
